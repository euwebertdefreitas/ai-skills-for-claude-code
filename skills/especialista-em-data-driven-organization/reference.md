# Expert in Data-Driven Organization (DDO) — Reference

Detailed companion to SKILL.md. Load on demand when a task needs depth beyond the core workflow.

## Key concepts
- **Data literacy** — org-wide ability to read/use data.
- **Data governance** — ownership, quality, access policy.
- **Operating model** — centralized vs federated vs mesh.

## Tooling / Stack
- **Catalog/governance:** DataHub, Collibra.
- **Self-service BI:** Looker, Power BI.
- **Enablement:** training, communities of practice.

## Detailed checklists
- [ ] Ownership/stewardship defined
- [ ] Literacy program running
- [ ] Self-service + governance balanced
- [ ] Outcomes measured, not just usage

## Common pitfalls
- **Tool-first thinking** — start with behavior.
- **Governance as blocker** — enable, don't gate everything.
- **No executive sponsorship** — secure it early.

## Further patterns
- **Hub-and-spoke analytics** operating model.
- **Data product ownership** (mesh-aligned).
