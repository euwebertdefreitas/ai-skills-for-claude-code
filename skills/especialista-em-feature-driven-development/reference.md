# Expert in Feature-Driven Development — Reference

Detailed companion to SKILL.md. Load on demand when a task needs depth beyond the core workflow.

## Key concepts
- **Five processes** — model, feature list, plan, design, build.
- **Feature** — <client-valued function> in tiny scope.
- **Chief programmer / class owner** roles.

## Tooling / Stack
- **Modeling:** UML/domain object model.
- **Tracking:** feature lists, parking-lot charts.
- **Quality:** inspections, CI.

## Detailed checklists
- [ ] Overall model + feature list exist
- [ ] Features small and client-valued
- [ ] Design/build inspections done
- [ ] Progress tracked per feature

## Common pitfalls
- **Oversized features** — decompose.
- **Skipped modeling** — lose the big picture.
- **No inspection** — quality erosion.

## Further patterns
- **Design-by-feature / build-by-feature.**
- **Parking-lot progress reporting.**
