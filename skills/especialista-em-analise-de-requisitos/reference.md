# Expert in Requirements Analysis — Reference

Detailed companion to SKILL.md. Load on demand when a task needs depth beyond the core workflow.

## Key concepts
- **Functional vs non-functional** — behavior vs quality attributes.
- **Acceptance criteria** — testable definition of done.
- **Traceability** — need ↔ requirement ↔ test.

## Tooling / Stack
- **Elicitation:** interviews, workshops, prototypes.
- **Specification:** user stories, use cases, BDD.
- **Management:** traceability matrix.

## Detailed checklists
- [ ] Need behind each requirement clear
- [ ] Acceptance criteria defined
- [ ] Non-functional captured
- [ ] Validated and traceable

## Common pitfalls
- **Gold-plating** — scope creep.
- **Ambiguity** — quantify quality attributes.
- **Missing stakeholders** — incomplete needs.

## Further patterns
- **User stories + acceptance criteria.**
- **MoSCoW prioritization.**
